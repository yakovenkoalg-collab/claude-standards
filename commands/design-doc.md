Создай или обнови дизайн-документ для задачи. Аргументы: $ARGUMENTS

## Использование
```
/design-doc #50 "Модель трёх горизонтов"
/design-doc "SSO role override"
```

## Контекст
Дизайн-док — обязательный артефакт перед началом реализации нетривиальной задачи. Он фиксирует решения, выявляет стыки с существующим кодом и предотвращает «баги по умолчанию».

## Шаги

### 1. Изучи контекст
- Прочитай существующие дизайн-доки в `docs/design/` (если есть) для понимания формата
- Изучи текущий код, затронутый задачей
- Если указан номер задачи — найди описание в roadmap

### 2. Заполни обязательные секции

```markdown
# {Название задачи}

## Цель
Одно предложение: что пользователь получит в результате.

## Инвентаризация данных
ВСЕ места, где затронутая сущность читается/пишется:
- SSR page.tsx (какие)
- API routes (какие)
- Client-side refresh (какие)
- Stores/contexts (какие)

## Карта компонентов
ВСЕ существующие компоненты, использующие затронутые данные.
Для каждого: используют prop напрямую или через useState(prop)?

## Алгебра фильтров
Для страниц с фильтрами:
- Дефолтные значения каждого фильтра
- Не скрываются ли новые данные дефолтами?

## Модель данных
Изменения в Prisma schema (новые модели, поля, enum'ы).
Миграции.

## API-контракт
Новые/изменённые эндпоинты с примерами request/response.

## UI-контракт
Для каждого нового компонента:
- Пустые данные
- Loading состояние
- Ошибка API
- Dark theme
- Мобильный вид
- Accessibility (роли, клавиатура)

## Стыки (файлы для изменения)
Явный список СУЩЕСТВУЮЩИХ файлов, которые нужно ИЗМЕНИТЬ.
Не только новые файлы — именно стыки опаснее всего.

## Чеклист интеграции
- [ ] Все data sources синхронизированы (SSR + API + refresh)
- [ ] useState(prop) → useEffect sync где нужно
- [ ] Фильтры не скрывают новые данные
- [ ] Dark theme проверен
- [ ] Мобильный вид проверен
- [ ] Документация обновлена (docs-context.ts, docs-content.ts)
- [ ] OpenAPI схема зарегистрирована
- [ ] Миграция создана
```

### 3. Покажи пользователю
Выведи дизайн-док для ревью. Дождись подтверждения или правок.

### 4. Сохрани
Сохрани в `docs/design/{номер-или-slug}-design.md`.
Если есть `data/roadmap.json` — обнови поле `designDoc` у задачи.
