Аудит Zod-валидационных схем на соответствие Prisma-модели. Аргументы: $ARGUMENTS

## Контекст

В проектах на Prisma + Zod частая проблема: поле в Prisma nullable (`String?`), а в Zod Create-схеме только `.optional()` без `.nullable()`. Это приводит к ошибке валидации, когда фронтенд отправляет `null`.

## Шаги

### 1. Определи проект
Если проект указан в аргументах — используй его. Иначе определи по текущей рабочей директории. Убедись, что в проекте есть `prisma/schema.prisma` и каталог с Zod-схемами (обычно `lib/api-schemas/models/` или `lib/schemas/`).

### 2. Прочитай Prisma-схему
Прочитай `prisma/schema.prisma`. Для каждой модели составь карту полей:
- Имя поля
- Nullable: `Type?` → да, `Type` → нет
- Default: `@default(...)` → да
- Relation: `@relation(...)` → пропустить (не скалярное поле)

### 3. Прочитай все Zod-схемы
Найди все файлы с Zod-схемами. Для каждой **Create**-схемы проверь каждое поле:

**Категория 1 — Prisma nullable, Zod не nullable:**
Поле `Type?` в Prisma, но в Zod только `.optional()` без `.nullable()`.
Фронтенд может отправить `null`, Zod отклонит.
Фикс: добавить `.nullable()` перед `.optional()`.

**Категория 2 — Prisma required (без nullable, без default), Zod optional:**
Поле обязательно в БД, но Zod пропустит `undefined`.
Риск: Prisma упадёт при INSERT.
Фикс: убрать `.optional()` или добавить `.default()`.

**Категория 3 — Prisma с default, Zod required:**
Поле имеет `@default()` в Prisma, но Zod требует его обязательно.
Нет ошибки, но лишнее ограничение для клиента.
Рекомендация: добавить `.optional()`.

### 4. Покажи результат
Выведи таблицу найденных проблем:
```
| Файл | Строка | Поле | Категория | Prisma | Zod | Фикс |
```

Спроси пользователя: **«Исправить автоматически?»**

### 5. Исправь (если подтверждено)
Для категории 1: добавь `.nullable()` перед `.optional()`.
Для категории 2: предупреди, не исправляй автоматически (может быть намеренно).
Для категории 3: предложи, но не навязывай.

После исправлений запусти `npx tsc --noEmit` для проверки.

### 6. Update-схемы
Повтори анализ для **Update**-схем. Обычно они уже корректны (`.nullable().optional()`), но проверь на всякий случай.
