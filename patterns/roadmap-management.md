# Паттерн: Управление Roadmap

> Единый подход к ведению бэклога, роадмапа и отслеживания прогресса в проектах.

## Структура данных

### roadmap.json (единый источник правды)
```json
{
  "meta": {
    "project": "moex-portal",
    "lastUpdated": "2026-04-02",
    "version": "0.31.0"
  },
  "waves": [
    {
      "id": "1",
      "name": "Ядро",
      "description": "MVP: команды, сотрудники, цели"
    }
  ],
  "tasks": [
    {
      "id": "1",
      "title": "CRUD команд",
      "wave": "1",
      "status": "DONE",         // OPEN | DONE | CANCELLED | MERGED
      "version": "0.1.0",       // в какой версии выполнено
      "priority": "HIGH",       // CRITICAL | HIGH | MEDIUM | LOW
      "type": "business",       // business | tech
      "designDoc": "teams-design.md",
      "dependencies": ["2"],    // блокирующие задачи
      "mergedInto": "1-2",      // только для MERGED — id объединённой задачи
      "description": "..."
    }
  ]
}
```

### roadmap.md (человекочитаемая версия)
**Автогенерируется** из `roadmap.json` скриптом `scripts/generate-roadmap-md.ts`.
Содержит:
- Сводную таблицу волн с прогрессом
- Детальные таблицы задач по волнам
- Открытые задачи с блокерами
- Таблицу дизайн-документов

### version.ts (changelog)
```typescript
export const CHANGELOG = [
  { version: "0.29.4", date: "2026-04-01", changes: [
    "[Новое] Описание с точки зрения пользователя",
    "[Улучшение] Что стало лучше",
    "[Исправление] Что починили",
  ]},
];
```

## Правила ведения

### Статусы задач
| Статус | Когда |
|--------|-------|
| `OPEN` | Задача в бэклоге, не начата |
| `DONE` | Вошла в релиз, обязательно поле `version` |
| `CANCELLED` | Отменена (с указанием причины в description) |
| `MERGED` | Поглощена другой задачей, обязательно поле `mergedInto` |

### Волны
- Волна = логическая группа связанных задач
- Волна `DONE` когда все задачи DONE или CANCELLED
- Новые задачи добавляются в существующую волну или создают новую
- Волны нумеруются последовательно

### Зависимости
- `dependencies: ["#2", "#5"]` — задача блокирована до завершения #2 и #5
- При завершении задачи — проверить, разблокировались ли зависимые
- Цикличные зависимости запрещены

### Релизный цикл
При `/release`:
1. Обновить `meta.version` и `meta.lastUpdated`
2. Задачи `OPEN` → `DONE` (если вошли в релиз) + указать `version`
3. Добавить новые задачи, если они реализованы, но отсутствуют в `tasks[]`
4. Запустить `npx tsx scripts/check-roadmap-sync.ts` (CI-валидация)
5. Запустить `npx tsx scripts/generate-roadmap-md.ts` (автогенерация roadmap.md)

### Стиль описания changelog
- Писать **с точки зрения пользователя**: что он теперь может делать
- Категории: `[Новое]`, `[Улучшение]`, `[Исправление]`
- Без технических деталей: не упоминать API-пути, библиотеки, параметры URL
- Технологические доработки описывать, только если значимы для пользователя

## UI-компонент (рекомендуемый)

### About / Changelog страница
```
/about
  Tabs: Changelog | Дорожная карта

  Changelog:
    Карточки по версиям, фильтр по категории (Новое/Улучшение/Исправление)

  Дорожная карта:
    Фильтры: волна, статус, теги
    Таблица задач с цветовой индикацией статуса
    Секция "Что дальше" — ближайшие OPEN задачи
```

### Данные
- Changelog: `lib/version.ts` → `CHANGELOG[]`
- Roadmap: `data/roadmap.json` → загружается в SSR
